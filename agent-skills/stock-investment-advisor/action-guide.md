# 可操作投资建议指南

用户要的是**能落地的近期建议**（看几日线、什么价位考虑、什么条件才介入），不是空泛「观望」或让用户自己去查别的命令。

---

## 禁止「踢皮球」

输出中**不得**出现：

- 「建议用 `/stock-analyze` 进一步分析」
- 「可自行查看 K 线 / 资金面」而不给结论
- 仅写「观望 / 合理」且**无任何**价位、均线、条件
- 把多个工具原始 JSON 贴出来让用户自己读
- 「必涨 / 必买 / 稳赚 / 全仓」

**必须**在「近期操作建议」中给出：周期 + 倾向 + 条件 + 失效位（有数据时写具体数字）。

---

## 必拉技术数据（分析类请求）

除 advisors.md 清单外，**一律补充**：

```bash
python scripts/em.py get_historical_series --secid {secid} --limit 60 --indicators ma
python scripts/em.py get_kline --secid {secid} --limit 20
```

从结果提取（最新一根）：

| 字段 | 用途 |
|------|------|
| 现价 | 对比均线 |
| MA5 / MA20 / MA60 | 短/中趋势 |
| 近 20 日最高、最低 | 压力 / 支撑参考 |
| 相对沪深300 | 强弱 |

可选：`get_short_term_monitor`（官方涨停基因）、`get_indicator_interpretation`（信号概率）。

## 报告模板（增强版 · 必填）

```markdown
# {名称} · {顾问名} 投资分析

> 顾问：{id} | 数据截止：{时间} | 持有周期视角：{短线5~20日 / 波段1~3月 / 中长期}

## 一、一句话结论

**总立场**：{低估|合理|高估|暂不介入|可跟踪} · **近期操作倾向**：{可分批试探|等回调再评估|持有观望|持有者考虑减仓|数据不足}

{用 2 句话说明：现在处在什么位置、近期最该盯什么}

## 二、近期操作建议（核心）

| 项目 | 建议 |
|------|------|
| **看线周期** | 以 {日K / 日K+20日线 / 周K} 为主；{若短线} 辅以 5 日线 |
| **操作倾向** | {见下表用语} |
| **理想关注/介入区间** | {有 MA/支撑则写 XX~XX 元；无则写「待回踩 20 日线附近」并标注估算 MA 价} |
| **确认条件** | 例：放量站稳 20 日线 / 突破近 20 日高点 XX 且主力 3 日净流入 / 财报后 PEG 确认 |
| **回避情形** | 例：不追高于 XX（前高/MA20 上方 X%）；破 XX 则短期逻辑失效 |
| **计划持有观察期** | {5~10 个交易日 / 1~3 个月}，到期重新评估 |

### 操作倾向用语（合规）

| 倾向 | 含义 |
|------|------|
| **可分批试探** | 质量/趋势尚可，控制仓位，分 2~3 次在关注区间内布局 |
| **等回调再评估** | 基本面 OK 但价格偏贵/短期过热，不追，等到区间或条件满足 |
| **持有观望** | 已持有者继续观察；未持有者暂不新建仓 |
| **暂不介入** | 基本面或趋势有一项明显不达标 |
| **持有者考虑减仓** | 估值或趋势显著恶化，非「必须卖」 |

禁止写「强烈买入 / 清仓 / 必涨」。
```
