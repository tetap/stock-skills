# 多重审核协议（报告输出前强制执行）

**结构化入口**：MCP/CLI 调用 `get_review_protocol(flow=B|C|D)` 获取轮次、门禁与 §7 模板路径。

**核心原则**：报告不是「拉数 → 写一遍 → 交差」。必须经历 **数据审计 → 魔鬼质疑 → 一致性测试 → 压力测试 → 置信度门禁** 多轮打击，**未过门禁不得输出终稿**。

适用：**B 个股全量**（完整 6 轮）、**C 板块**（精简 4 轮）、**D 热点**（精简 3 轮）。

---

## 置信度门禁（全局）

| 分数 | 含义 | 能否出终稿 |
|------|------|------------|
| **≥7** | 数据较全、矛盾已解释、立场可辩护 | ✅ 可输出 |
| **5~6** | 有缺口但主逻辑成立 | ✅ 可输出，评级不得高于「右侧等待」，§7 须列未解问题 |
| **≤4** | 数据不足或矛盾未解 | ❌ **禁止定稿**；补拉工具或降级为「数据不足，仅列事实」 |

**初始置信度不得高于 6**；只有过完 R4+R5 且无未解 P0 矛盾，才可升到 7+。

---

## B 流程：个股六轮审核

### R0 · 数据采集（前置）

- 工具 ≥ **20 次**（含 `get_quant_technical`）
- 缺项记入 `gaps[]`，不得假装有数

### R1 · 初稿（内部，不直接给用户）

按 [analysis-report.md](analysis-report.md) 写 **Draft v0**，含：评级、四维度表、交易计划、quant_verdict。

### R2 · 数据审计员（Audit）

**角色**：只认工具返回值，不认叙事。

必做清单：

- [ ] §1 每个数字能否指向具体工具字段？（PE←valuation，主力←fund_flow…）
- [ ] 四维度是否各 ≥1 个硬指标？
- [ ] `get_quant_technical` 的 5d/60d/158 是否都已写入 §3？
- [ ] 新闻是否 ≥2 条个股 + 1 条行业热点？来源是否标注？
- [ ] 有无「未获取到 XX」被隐瞒？

**输出（内部）**：

```markdown
### R2 审计
- 通过项：…
- 缺口：…
- 幻觉风险：…（无则写无）
- 处置：补拉 {工具} / 删去无依据句
```

有 **P0 缺口**（缺现价、缺财报、缺资金）→ 回到 R0 补拉，不得进入 R3。

### R3 · 魔鬼代言人（Challenge）

**角色**：专门推翻 Draft 的评级与交易计划。

至少提出 **5 个尖锐问题**，必须覆盖：

1. **基本面**：业绩/估值是否支持当前市值？有没有价值陷阱？
2. **技术面**：quant 短长分歧是否在粉饰？MA 与 quant_verdict 是否打架？
3. **资金面**：主力流出是否否定反弹叙事？
4. **情绪面**：热点是否一日游？新闻与股价是否脱节？
5. **交易计划**：买点是否已破/失效？止损是否过窄/过宽？

**规则**：

- 每个问题必须 **用数据回答** 或 **承认无法反驳并下调置信度**
- 若 3 个以上问题无法反驳 → 评级 **降一档**（如积极跟踪→右侧等待）

```markdown
### R3 魔鬼质疑
| # | 质疑 | 数据回应 | 是否削弱 thesis |
|---|------|----------|-----------------|
| 1 | … | … | 是/否 |
```

### R4 · 一致性测试（Cross-check）

构建 **一致性矩阵**，不允许矛盾裸奔：

| 信号源 A | 信号源 B | 一致？ | 不一致时采信谁、理由 |
|----------|----------|--------|----------------------|
| Alpha158 | Alpha360 5d/60d | | |
| quant_verdict | MA20/60 位置 | | |
| 主力 5 日 | 股价 5 日 | | |
| 财报趋势 | 估值 PE/PB | | |
| 板块强弱 | 个股相对沪深300 | | |

**硬规则**：

- 158 与 360 **同向偏空** + 资金流出 → 不得给「积极跟踪」
- 360 **短线改善·中线仍弱** → 评级最高「事件驱动博弈」，且须写观察期 ≤10 日
- 基本面 **明确回避级**（亏损+极高 PB 且无催化）→ 技术面短线多也只能「博弈」，不能「配置」

### R5 · 压力测试（Stress）

模拟 **3 个失败场景**，每个 1~2 句对股价/逻辑的影响：

1. **催化剂落空**（财报不及预期 / 政策未落地）
2. **市场系统性下跌**（沪深300 -3%）
3. **逻辑被证伪**（§1「若我错了」里的触发条件发生）

若压力测试下 **止损位守不住或逻辑全崩** → 置信度 −1，交易计划须收紧。

### R6 · 置信度门禁 + 终稿

1. 汇总 R2~R5，计算 **最终置信度 X/10**（写明加减分项）
2. 若 X≥7：输出 **终稿 + §7 审核纪要**
3. 若 X=5~6：终稿 + 明确「未解问题」+ 评级上限
4. 若 X≤4：只输出事实摘要 + 缺口清单，**不给买卖区间**

---

## 终稿必含：§7 审核纪要

**给用户可见**，证明报告经过多轮打击（简洁，8~15 行）：

```markdown
## 7. 审核纪要

> 终稿置信度：**{X}/10**（初稿 {Y}/10 → 经 {N} 轮修订）

| 轮次 | 关键质疑/测试 | 结论 | 对结论影响 |
|------|---------------|------|------------|
| R2 数据审计 | … | 通过/补数 | … |
| R3 魔鬼质疑 | … | … | 评级从 A→B |
| R4 一致性 | 158 vs 360 分歧 | … | … |
| R5 压力测试 | 财报不及预期 | … | 置信度 +0/-1 |

**仍存不确定性**：{0~2 条，无则写「无重大未解矛盾」}
```

## C 流程：板块四轮（精简）

| 轮次 | 内容 |
|------|------|
| R0 | 板块工具 ≥12 + 候选股轻扫 |
| R2 | 审计：板块 K 线、资金、新闻是否齐 |
| R3 | 质疑：为何选这几只而非涨幅榜？退潮风险？ |
| R4 | 一致性：板块涨 + 个股资金是否同向 |
| R6 | 门禁 + §7 审核纪要（可 5 行） |

---

## D 流程：热点三轮（精简）

| 轮次 | 内容 |
|------|------|
| R0 | 快讯 + 板块 + 资金 |
| R3 | 质疑：热点持续性？是否已 price in？ |
| R4 | 交叉：新闻主题 vs 板块涨幅 vs 资金 |
| R6 | 主线排序置信度 + 2 条「若错了」 |

---

## Agent 执行纪律

1. **禁止**跳过 R2~R5 直接给用户终稿（查价/ list 除外）
2. **禁止**初稿置信度 >6
3. **禁止**审核纪要写「无」却未展示任何质疑过程
4. 可在同一回复内用 **折叠/小标题** 呈现 R2~R5，但 **§7 审核纪要必须出现在终稿**
5. 有 MCP 时优先用工具复核；质疑中发现缺口 **立即补拉** 再修订

## 与 quant_verdict 的衔接

| quant_verdict.summary | 评级上限 | R3 必问 |
|----------------------|----------|---------|
| 量化偏多 | 积极跟踪 | 是否过热？资金是否配合？ |
| 短线改善·中线仍弱 | 事件驱动博弈 | 5 日后是否衰竭？158 为何未转多？ |
| 反弹脉冲·因子未跟上 | 事件驱动博弈 | 是否一日游？ |
| 量化偏空 | 明确回避 / 右侧等待 | 有无错杀？催化剂能否对冲？ |
| 量化中性 | 右侧等待 | 等什么信号？ |
