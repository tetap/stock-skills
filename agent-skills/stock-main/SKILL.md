# 主命令编排（/stock）

**原则**：一个入口 `/stock`。先 **尽量拉全数据** → 再写 **有立场、可执行** 的研究报告（专业口吻，明确评级，禁止空泛观望）。

## 报告风格（重要）

- **专业**：每个判断锚定数据；四维度交叉（基本面/技术/资金/情绪）；相对沪深300与板块强弱。
- **大胆**：§1 必须给 **评级**（积极跟踪/右侧等待/明确回避/事件驱动博弈）+ **置信度 X/10** + **具体买卖区间**。
- 允许写「明确回避」「不宜新建仓」；禁止「必涨必买」和没有价位的「建议关注」。

## 意图路由

| 意图 | 触发 | 动作 |
|------|------|------|
| 查现价 | 仅代码/名称，或「多少钱、现价」且无分析词 | → **stock-quick-lookup** |
| **个股分析** | 分析某**具体股票**、能不能买、看几日线 | → **B 流程** |
| **板块/选股** | 板块、行业、概念 + 走势/看好/推荐/龙头/有哪些票 | → **C 流程** |
| **热点/情绪** | 热点、情绪、今天什么板块、市场怎么样、新闻 | → **D 流程** |
| list | list/列表 | → 说明：`/stock 分析 股票名` |

**不再使用** buffett/graham/lynch 等顾问 ID。

---

## B 流程：个股全量分析

1. `resolve_symbol` → secid、code、name
2. 拉数 ≥18 次，见 [analysis-report.md](analysis-report.md)（**含新闻/情绪**）
3. 输出 6 节报告（见 analysis-report.md），§1 含 **评级、置信度、交易计划、若我错了**

---

## C 流程：板块走势 + 选股

**示例**：`/stock 帮我看看电池板块最近的走势，给我几个你看好的股票`

1. 读 [sector-report.md](sector-report.md)
2. `search_sectors("电池")` → 选定板块
3. 拉板块 K 线、资金、成分股 + **市场快讯/要闻（带 keyword）**
4. 对 3~5 只候选做轻量扫描（现价、估值、资金、新闻）
5. 输出板块报告 + 首选标的表（**按优先级排序**，含买点/失效位）

---

## D 流程：市场热点与情绪

**示例**：`/stock 今天有什么热点` / `最近情绪面怎么样`

1. 读 [market-brief.md](market-brief.md)
2. 拉 7×24 快讯、要闻、板块排行、大盘/个股资金
3. 交叉提炼 **主线排序**（2~4 条带优先级），给可跟踪/回避条件

**雪球帖子**：若需讨论正文且出现 `xueqiu_auth_hint`，先引导用户登录雪球并从 Cookie 复制 `xq_a_token` 到 `XUEQIU_TOKEN`，用户确认后再继续。

## 输出禁令

- 禁止投资顾问/流派章节
- 禁止只拉 2 个工具就写报告
- 禁止 redirect 其他命令
- 禁止必涨必买
- 禁止 §1 只有「观望/关注」而无评级、价位、置信度
